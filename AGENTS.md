## Setup

```bash
# Backend
cd backend
uv sync
cp .env.example .env       # fill in keys
uv run uvicorn app.main:app --reload --port 8000

# Frontend
cd frontend
pnpm install
echo "VITE_API_BASE_URL=http://localhost:8000" > .env.local
pnpm dev
```

System dependency: **ffmpeg** must be on `PATH`.

## Code conventions

### Python (backend)

- `from __future__ import annotations` at the top of every file
- Type hints everywhere — `mypy --strict`-ready in spirit
- `pydantic` models for **all** API I/O and all third-party API responses we care about
- Prefer functions over classes when a function works. `JobStore` is the only stateful class needed for MVP.
- `httpx.AsyncClient` for all HTTP — never `requests`
- `asyncio.gather` for parallel external calls (Hera ×4, agent ×4)
- `ruff` for formatting and linting (`line-length = 100`)

### TypeScript (frontend)

- `strict: true`, no `any`
- Tailwind utility classes only — **no custom CSS files**, no CSS modules
- Component files PascalCase (`Hero.tsx`, `JobProgress.tsx`)
- Hooks under `src/hooks/`, components under `src/components/`, pages under `src/pages/`
- `fetch` directly — no axios, no react-query for MVP
- No state management library — `useState` + Context is enough

### Comments

- Default to writing none.
- Add a one-line comment only when the *why* is non-obvious — a hidden constraint, a workaround, a third-party quirk.
- Never write "this function does X" docstrings; identifier names should already say that.

### Errors

- Validate at boundaries: API request bodies, third-party responses, file uploads.
- Trust internal calls — don't add defensive `if x is not None` everywhere.

## Architecture rules — do not violate

These are intentional constraints for the MVP. If a task pressures you to break one, raise it with the user instead of working around it.

1. **No database.** Job state is an in-memory `dict` in the FastAPI process. If you feel the urge to add Postgres / Redis / SQLite, stop.
2. **No queue.** Pipelines run via `asyncio.create_task` from the request handler. No Celery, RQ, or external queue.
3. **No persistent file storage.** Intermediate files in `/tmp/{job_id}/`. Final MP4 streamed to browser. No S3, no Vercel Blob.
4. **No auth.** Public demo. No accounts, no API keys for end users.
5. **Server-side third-party keys only.** Gemini, Tavily, ElevenLabs, and Hera keys live in backend `.env`. Frontend talks to backend, never to a third-party directly.
6. **Single backend process.** No worker / web split. Don't introduce one.

## The product's AI agents

Three AI agents run **inside the product**. They are not you, the coding agent. Do not put their prompts here — put them in `docs/agent-prompts.md`.

| Agent          | Job                                                          | Input                          | Output                              |
| -------------- | ------------------------------------------------------------ | ------------------------------ | ----------------------------------- |
| **Finance**    | Extract top 6–10 facts from a business document              | Cleaned PDF text               | Markdown Q&A list                   |
| **Scripter**   | Turn the Q&A into a 4-scene voiceover script                 | Q&A markdown                   | JSON `{ title, scenes[4] }`         |
| **Hera** (×4)  | Convert one scene + audio timings into a Hera animation spec | Scene + sentence timings       | Hera-format JSON                    |

All three use Google Gemini 2.5 Pro. Full system prompts: [docs/agent-prompts.md](docs/agent-prompts.md). Pipeline: [README.md](README.md).

## Definition of done for a task

- The corresponding `TASK.md` checkbox is ticked in the same change.
- Code runs locally without errors.
- For the full pipeline: a sample PDF produces a playable MP4 in under 3 minutes.
- No new dependencies introduced for trivial reasons.
- No new files created when an existing file would do.
